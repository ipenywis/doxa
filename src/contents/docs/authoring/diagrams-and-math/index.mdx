## Math Examples

Inline math is useful for small formulas like $$queryWords \ge 3$$.

Use block math for scoring and ranking explanations:

```math
relevance = \frac{title + headings + keywords + content}{\log(length + 1)}
```

```math
latencyBudget = renderTime + searchTime + modelTime
```

<Note title="Diagram budget">
  Diagrams should explain relationships that are hard to scan in prose. If a
  list or table is clearer, use a list or table.
</Note>

## Mermaid Patterns

| Diagram type     | Use it for                                                  |
| :--------------- | :---------------------------------------------------------- |
| Flowchart        | Architecture, routing, deployment, and content pipelines.   |
| Sequence diagram | AI calls, request lifecycles, and handoffs between systems. |
| ER diagram       | Data models and relationships.                              |
| Decision tree    | Configuration choices and troubleshooting paths.            |

## Practical Rule

One good diagram can anchor a complex page. Several decorative diagrams make the page slower to read.
