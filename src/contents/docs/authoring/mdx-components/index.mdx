Doxa registers a focused component set for documentation pages in `src/lib/components.ts`. These components are available directly in MDX without per-page imports.

## Component Catalog

| Component                        | Use it for                                                       |
| :------------------------------- | :--------------------------------------------------------------- |
| `CardGrid` and `Card`            | Navigation, feature groups, related pages, and external links.   |
| `Note`                           | Important callouts, prerequisites, warnings, and success states. |
| `Step` and `StepItem`            | Sequential workflows.                                            |
| `Tabs`                           | Alternative commands, providers, platforms, or languages.        |
| `Mermaid`                        | Architecture diagrams, agent flows, and process charts.          |
| `FileTree`, `Folder`, and `File` | Project structure and content layout.                            |

## Notes

Use notes for information a reader should not miss.

<Note title="Default note">
  Use the default note for neutral context, requirements, and reminders.
</Note>

<Note title="Success" type="success">
  Use success notes to confirm expected output or completion states.
</Note>

<Note title="Warning" type="warning">
  Use warnings for configuration details that can break a workflow.
</Note>

<Note title="Danger" type="danger">
  Use danger notes for irreversible actions, security risks, or destructive
  commands.
</Note>

## Steps

Use `Step` for workflows that must happen in order.

<Step>
  <StepItem title="Write the MDX">
    Add frontmatter and body content under `src/contents/docs`.
  </StepItem>

<StepItem title="Update navigation">
  Add the page to `src/contents/settings/documents.json`.
</StepItem>

  <StepItem title="Regenerate search data">
    ```bash
    pnpm generate-content-json
    ```
  </StepItem>
</Step>

## Tabs

Tabs are useful when the user chooses one option from a small set.

<Tabs defaultValue="pnpm" className="pt-5 pb-1">

  <TabsContent value="npm">```bash npm install ```</TabsContent>

## Composition Rules

- Put cards in grids; do not place standalone cards between paragraphs.
- Use steps for sequential work, not feature lists.
- Keep notes short enough to scan.
- Use tabs only when all tab values are mutually exclusive.
- Prefer one Mermaid diagram per page section.
