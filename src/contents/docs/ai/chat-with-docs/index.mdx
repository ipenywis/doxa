Chat with Docs gives readers a side panel where they can ask questions about the documentation they are reading. The assistant is constrained to the docs corpus and uses tools to search and read MDX before answering.

## Reader Experience

The chat button appears in the navigation on `/docs` routes when `Settings.features.ai.chat` is enabled. The drawer supports:

- Streaming assistant responses.
- Visible tool-call activity.
- Expandable tool output.
- Conversation history stored in the browser.
- New conversation and delete controls.
- Expanded and compact drawer widths.

<Note title="Docs-only answers">
  The system prompt tells the assistant to answer only from available
  documentation files. If a question is outside the docs, it should say so.
</Note>

## Enable Chat

<Step>
  <StepItem title="Turn on the feature flag">
    ```ts {4-8} showLineNumbers
    export const Settings = {
      features: {
        tableOfContents: true,
        ai: {
          chat: true,
          chatWithPage: true,
          floatingInput: true,
        },
      },
    }
    ```
  </StepItem>

  <StepItem title="Choose a provider">
    Configure `src/settings/ai.ts`.

    ```ts showLineNumbers
    export const aiConfig = {
      provider: "openrouter",
      model: "z-ai/glm-4.7:nitro",
      codeSnippets: true,
      maxContextChunks: 12,
      maxResponseTokens: 2048,
    }
    ```
  </StepItem>

  <StepItem title="Set the API key">
    ```bash
    AI_API_KEY=your-provider-key
    ```
  </StepItem>

  <StepItem title="Ask a docs question">
    Try a question that exists in the docs, such as "How do I switch Doxa to GitHub content?"
  </StepItem>
</Step>

## Failure Modes

| Symptom | Likely cause | Fix |
| :-- | :-- | :-- |
| Chat returns a missing key error | `AI_API_KEY` is not set | Add the provider key to the environment. |
| Answers miss new pages | Search data is stale or docs were not deployed | Regenerate search data and redeploy. |
| The assistant cannot cite a topic | The topic is not documented | Add or expand the relevant MDX page. |
| Tool calls show no matches | Query terms do not appear in docs | Improve headings, descriptions, and keywords. |

<Note title="Production guidance" type="warning">
  Keep provider keys on the server. Do not expose `AI_API_KEY` as a Vite public
  environment variable.
</Note>
