To install and edit the Documents, you need to have several prerequisites in place. Here's a list of all the essential pre-requisites
for setting up and working on this project.

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fipenywis%2Fdoxa-docs-template&project-name=doxa-docs-template&repository-name=doxa-docs-template)

## Prerequisites

<Step>
<StepItem title="System Requirements">

- **Node.js:** Make sure you have Node.js installed. The project uses `pnpm`, which comes with Node.js. Install Node.js from here.
- **NPM, PNPM or Yarn:** Since the project uses `pnpm` commands, ensure that pnpm is installed with Node.js. Alternatively, you can use Yarn if preferred.

To verify installation, run:

```bash
node -v
pnpm -v
```

</StepItem>
<StepItem title="Git">
You need to have Git installed to clone the repository. Download Git from [here](https://git-scm.com/).

```bash
git --version
```

</StepItem>
<StepItem title="Code Editor">
You’ll need a code editor to work on the project. Common options include:

- [Visual Studio Code](https://code.visualstudio.com/)
- [WebStorm](https://www.jetbrains.com/webstorm/)
- Any other text editor you prefer.

</StepItem>
<StepItem title="Required Knowledge">
- **TanStack Start:** The app uses TanStack Start on top of Vite and React Router.
- **React:** The project uses React, so understanding React's component structure and hooks will help.
- **TypeScript:** The project is written in TypeScript, so you should have a basic understanding of it.
- **Tailwind:** The project uses Tailwind CSS for styling.
- **MDX:** Markdown + JSX (MDX) is used for writing documentation.

</StepItem>
</Step>

## Installation

<Step>
<StepItem title="Clone the Repository">
First, clone the repository to your local machine using Git:

```bash
git clone https://github.com/ipenywis/doxa-docs-template.git
```

Change the directory to the project folder:

```bash
cd doxa-docs-template
```

</StepItem>
<StepItem title="Install Dependencies">
Use npm to install all the required dependencies:

```bash
pnpm install
```

This will install all the dependencies listed in the package.json file, including:

- TanStack Start
- React
- TypeScript
- Tailwind CSS
- MDX components
- Other dependencies like ESLint, PostCSS, Husky, etc.

</StepItem>

<StepItem title="Running the Development Server">
Start the development server by running:

```bash
pnpm run dev
```

This will start the local server, and you can open your browser at:

```arduino
http://localhost:3000
```

</StepItem>

<StepItem title="Editing the Project">
  - **MDX Files:** Documentation content is stored in `src/contents/docs`.
  - **Components:** You can edit React components in `src/components/`.
  - **Tailwind:** Global styling lives in `src/styles/globals.css`.

To make any changes to the project, open the files in your editor and modify them as needed.

<Note title="Required" type="danger">
  The project's search functionality relies on the Husky's automation to build
  `search-data/documents.json` ensure git commit is performed to generate this
  file.
</Note>

</StepItem>
<StepItem title="Building for Production">
Once you’ve made your changes and want to build the project for production:

```bash
pnpm run build
```

This will generate a production-ready build of the app.

</StepItem>
<StepItem title="Running Production Server">
To run the production server:

```bash
pnpm run start
```

The production app will be available at:

```arduino
http://localhost:3000
```

</StepItem>
</Step>

## Optional

<Step>
<StepItem title="Deploying to Vercel">
This project includes native Vercel support. To deploy to Vercel:

- Run `vercel link` to connect the local directory to your Vercel project.
- Set `VITE_SITE_URL` to the public URL for your docs site.
- Set `AI_API_KEY` if the AI chat feature is enabled.
- Run `pnpm run deploy:vercel`.
</StepItem>
<StepItem title="Deploying to Cloudflare Workers">
Cloudflare remains the default deployment target:

- Add the required Wrangler secrets, including `AI_API_KEY` if chat is enabled.
- Run `pnpm run deploy`.
</StepItem>
<StepItem title="Linting and Pre-commit Hooks">
The project uses ESLint for linting and Husky for Git hooks. These tools are configured automatically, but you can customize them in `.eslintrc.json` and `.husky` directories.
</StepItem>
</Step>

## Important Information

The project's search functionality relies on the Husky's automation to build `search-data/documents.json` ensure git commit is performed to generate this file.
