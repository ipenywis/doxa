Doxa configuration is split by concern. Site and SEO defaults live in `src/settings/main.ts`, AI provider settings live in `src/settings/ai.ts`, and content backend selection lives in `src/settings/content.ts`.

## Site Settings

`Settings` controls public metadata, branding, analytics, and feature flags.

```ts {8-12,35-43} showLineNumbers
export const Settings = {
  site: {
    name: "doxa",
    url: "https://docs.example.com",
    description: "Beautiful, fast documentation that stays out of your way.",
    keywords: ["documentation", "mdx", "ai"],
    icon: "/favicon.svg",
  },
  features: {
    branding: true,
    rightSidebar: true,
    feedbackEdit: true,
    tableOfContents: true,
    scrollToTop: true,
    ai: {
      chat: true,
      chatWithPage: true,
      floatingInput: true,
    },
    copyPage: {
      markdown: true,
      rawText: true,
    },
    rawMarkdown: true,
    loadFromGithub: false,
  },
};
```

| Setting group           | Purpose                                        |
| :---------------------- | :--------------------------------------------- |
| `site`                  | Name, URL, description, keywords, and favicon. |
| `company`               | Footer company name and link.                  |
| `seo`                   | Open Graph image, alt text, and social handle. |
| `analytics`             | Google Tag Manager id and enablement flag.     |
| `features`              | Toggles for docs shell behavior.               |
| `openGraph` / `twitter` | Social metadata used by the root document.     |

## Environment Variables

| Variable            |       Required       | Used for                                                       |
| :------------------ | :------------------: | :------------------------------------------------------------- |
| `VITE_SITE_URL`     |      Production      | Canonical URLs, sitemap, Open Graph, and AI-native route URLs. |
| `AI_API_KEY`        | When chat is enabled | Provider authentication for Chat with Docs.                    |
| `CONTENT_SOURCE`    |       Optional       | Selects `vite` or `github`. Defaults to `vite`.                |
| `DOXA_GITHUB_OWNER` |    GitHub source     | Repository owner for runtime docs loading.                     |
| `DOXA_GITHUB_REPO`  |    GitHub source     | Repository name for runtime docs loading.                      |
| `DOXA_GITHUB_TOKEN` | Private GitHub docs  | Read-only token for private docs repositories.                 |

## Sitemap Generation

Production builds generate `/sitemap.xml` through TanStack Start. Doxa seeds
the sitemap from `src/contents/settings/documents.json`; the default Cloudflare
build also prerenders each docs page and keeps link crawling enabled for
internally linked pages.

<Note title="Host inference" type="info">
  `VITE_SITE_URL` is the preferred sitemap host. If it is not set, the build
  checks `DOXA_SITE_URL`, `SITE_URL`, `PUBLIC_SITE_URL`, `PUBLIC_URL`,
  `VERCEL_PROJECT_PRODUCTION_URL`, `VERCEL_URL`, `VERCEL_BRANCH_URL`,
  `CF_PAGES_URL`, `URL`, `DEPLOY_URL`, and `DEPLOY_PRIME_URL` before falling
  back to `https://docs.example.com`.
</Note>

## Feature Toggle Guidance

- Keep `ai.chat` enabled for demos that showcase the AI assistant.
- Disable `ai.chatWithPage` when readers should use only the main Chat with Docs drawer.
- Disable `ai.floatingInput` when the bottom floating prompt is too prominent for your docs.
- Use `copyPage.markdown` and `copyPage.rawText` to control page-level copy formats.
- Disable `rawMarkdown` if you do not want any docs page exposed as raw `.md` (see [Raw Markdown Routes](/ai/raw-markdown)).
- Disable `feedbackEdit` unless `GitHubLink.href` points at a real repository.
- Keep `tableOfContents` enabled for long reference pages.
- Use `loadFromGithub` as a product-facing toggle only after `CONTENT_SOURCE=github` is configured.
